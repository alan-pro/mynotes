# 1 创建表

### 创建商品 good_sku 表及优化字段类型

根据你提供的INSERT语句，我为你生成了对应的`goods_spu`表创建语句，包含**字段类型、注释、主键约束**，并适配MySQL/MySQL8.0的语法规范，可直接执行：

```SQL
-- 创建商品SPU表（标准化表结构，适配MySQL/MySQL8.0）
CREATE TABLE IF NOT EXISTS `goods_spu` (
  `id` BIGINT NOT NULL COMMENT 'SPU主键ID',
  `create_time` DATETIME(6) NOT NULL COMMENT '创建时间（微秒级）',
  `update_time` DATETIME(6) NOT NULL COMMENT '更新时间（微秒级）',
  `name` VARCHAR(255) NOT NULL COMMENT '商品名称',
  `sold_num` INT NOT NULL DEFAULT 0 COMMENT '销量',
  `fav_num` INT NOT NULL DEFAULT 0 COMMENT '收藏数',
  `comments_num` INT NOT NULL DEFAULT 0 COMMENT '评论数',
  `goods_desc` LONGTEXT COMMENT '商品详情（富文本/HTML）',
  `goods_pack` LONGTEXT COMMENT '包装清单',
  `postsale_service` LONGTEXT COMMENT '售后服务说明',
  `category1_id` INT NOT NULL COMMENT '一级分类ID',
  `category2_id` INT NOT NULL COMMENT '二级分类ID',
  `category3_id` INT NOT NULL COMMENT '三级分类ID',
  -- 主键约束（保证ID唯一）
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品SPU表';
```

### 关键说明

1. **字段类型适配**：
    
    1. `id`用`BIGINT`（兼容大数值ID，避免INT溢出）；
        
    2. `create_time/update_time`用`DATETIME(6)`（支持微秒级时间，匹配INSERT里的`10:28:45.978125`格式）；
        
    3. `goods_desc/goods_pack/postsale_service`用`LONGTEXT`（兼容长文本/HTML内容，避免VARCHAR长度不足）；
        
    4. 数值型字段（销量/收藏数等）加`DEFAULT 0`（保证默认值合规）。
        
2. **字符集与引擎**：
    
    1. `CHARSET=utf8mb4`（支持emoji、特殊字符，通用最优选择）；
        
    2. `ENGINE=InnoDB`（支持事务、外键，MySQL默认推荐引擎）。
        
3. **执行顺序**：
    
    1. 先执行上面的`CREATE TABLE`语句创建表；
        
    2. 再执行你提供的`INSERT`语句插入数据。
        

### 总结

1. 表结构核心字段：ID（主键）+ 时间字段 + 商品基础信息 + 分类ID + 长文本详情字段；
    
2. 字段类型优化：用`LONGTEXT`存富文本、`DATETIME(6)`存微秒级时间，适配实际数据场景；
    
3. 约束：主键保证ID唯一，数值字段默认值避免空值问题。
