## 1.3 实现cd dev进入目的文件夹

-  方法一：使用符号链接（软链接）- 推荐
在 `/home/ai` 目录下创建一个指向 `dev` 的软链接：

```bash
cd ~
ln -s /home/ai/dev devlink
```
然后你就可以用 `cd devlink` 进入 `dev` 文件夹了。

但如果你想像你说的直接用 `cd dev`，可以创建同名的链接：
```bash
cd ~
ln -s /home/ai/dev dev
```
**注意**：如果 `~/dev` 目录已经存在，这个命令会失败。你可以先检查：
```bash
ls -la ~ | grep dev
```


- 方法二：使用 CDPATH 环境变量（更强大）
设置 `CDPATH` 让 `cd` 命令在指定目录中搜索：

在 `~/.bashrc` 或 `~/.bash_profile` 中添加：
```bash
export CDPATH=.:/home/ai
```
然后重新加载配置：
```bash
source ~/.bashrc
```
之后无论你在哪里，只要输入 `cd dev`，系统就会自动在 `/home/ai` 目录下查找 `dev` 文件夹并跳转。

- 方法三：使用别名（alias）
在 `~/.bashrc` 中添加别名：
```bash
alias cddev='cd /home/ai/dev'
```
然后输入 `cddev` 就可以跳转。

---

**推荐使用方法二**（CDPATH），因为它最接近你想要的效果：无论在哪里，输入 `cd dev` 就能直接进入 `/home/ai/dev`（只要没有同名的当前目录干扰）。

```shell
cd ~          # 先回主目录
cd /mnt/c/Users/pro  # 去 Windows 目录
cd dev        # 现在应该能直接跳转到 /home/ai/dev
```

## 1.5 apt下载速度太慢
修改 sources.list（推荐，永久生效）

1. 备份原文件
```shell
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
```

2. 编辑 sources.list

```shell
sudo vim /etc/apt/sources.list
```

3. 替换为国内源（以清华源为例）

按 `dd` 删除所有内容，然后粘贴以下内容：

```shell
# 默认注释了源码镜像以提高 apt update 速度，如有需要可自行取消注释
deb http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-proposed main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse

# 以下是 Ubuntu 官方合作伙伴源（包含一些第三方软件）
deb http://archive.canonical.com/ubuntu/ focal partner

# Docker 阿里云官方源
deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu focal stable
```

**注意**：上面的 `focal` 是 Ubuntu 22.04 的代号。请根据你的 Ubuntu 版本替换：

| Ubuntu版本 | 代号     |
| -------- | ------ |
| 24.04    | noble  |
| 22.04    | jammy  |
| 20.04    | focal  |
| 18.04    | bionic |

查看你的版本：

```shell
lsb_release -c
```

## 1.5 Docker 的官方源失效
按下面的步骤彻底替换 Docker 源为阿里云镜像版

1. 进入配置文件
```shell
(venv) ai@pro:~/dev/xiaofu$ sudo nano /etc/apt/sources.list
```

2. 编辑
```shell
# ctrl+/：移动到最后一行
# 更换阿里云docker源
deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu focal stable

# ctrl+O保存，回车确定，ctrl+X退出
```

3. 下载并配置Docker GPG密钥（解决签名验证问题）

```Bash
# 下载阿里云镜像的Docker公钥并保存
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 确保密钥文件权限正确
sudo chmod 644 /usr/share/keyrings/docker-archive-keyring.gpg
```

4. 清理apt缓存并重新更新（验证修复效果）

```Bash
# 清理无效缓存（可选）
sudo apt clean
sudo rm -rf /var/lib/apt/lists/*

# 重新更新源（此时不会再报Docker源的错）
sudo apt update
```


# 2 安装mysql

1. 在WSL上安装MySQL
```shell
sudo apt update
sudo apt install mysql-server
```

2. 配置mysql

```shell
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf # 任意位置输入，因为这是绝对地址
```
对这段代码进行修改

- `bind-address = 127.0.0.1`：注释掉这一行，相当于`bind-address = 0.0.0.0`，显式指定监听所有 IP。
- `mysqlx-bind-address = 127.0.0.1`：注释掉这一行，不注释MySQL X DevAPI只能在本地访问。

3. 启动mysql
```shell
sudo service mysql start  # wsl中运行，可能需要手动启动服务
sudo service mysql stop
sudo service mysql restart

mysql -uroot -p -h 127.0.0.1或WSL上的ip地址  # 非root账号
sudo mysql -u root  # 无密码sudo方式登录
```

4. 添加远程访问用户
```shell
-- 这样我们创建了一个名为root的用户，密码为123456，并允许他从任何地方连接到数据库。

mysql> CREATE USER 'root'@'%' IDENTIFIED BY '12345';

-- 这一步是为了让刚才创建的用户拥有所有权限。

mysql> GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
`u_xiaofu@localhost` → 只能本地 socket，**普通用户经常连不上**
`u_xiaofu@%` → 走 TCP，**任何用户都能连，不需要 sudo**

-- 或者给root用户只分配xiaofu数据库权限
mysql> grant ALL on xiaofu.* to 'root'@'%'; 

-- 刷新权限，使得刚刚的操作生效

mysql> FLUSH PRIVILEGES;

-- 如果你想要把生成的用户给删掉

mysql> DROP USER 'root'@'%';
```
重启MySQL服务：`sudo service mysql restart`

5. 在Navicat上连接MySQL

首先，我们需要先找到WSL的IP地址，记得不是那个127.0.0.1哦

```
# 安装Linux网络工具
sudo apt install net-tools
# 查看WSL的IP地址
ifconfig

# 简单方法
hostname -I
```

然后，使用Navicat连接到MySQL，连接方式如下：

- 连接类型：MySQL
- 主机名/IP地址：WSL的IP地址
- 端口：3306
- 用户名：root
- 密码：12345（这个你刚刚设置了啥就用啥）
- 数据库：（空白）


**可能的错误**

1. **WSL 不使用 systemd**：所以 `sudo systemctl status mysql` 会报错 `System has not been booted with systemd as init system`，使用下面方式

```shell
sudo service mysql status
```

2. **启动脚本报错**：`su: warning: cannot change directory to /nonexistent: No such file or directory` 是因为 MySQL 运行时需要的家目录 `/nonexistent` 不存在，这是一个常见的权限 / 配置问题。

	-  修复 `/nonexistent` 目录问题
	这个目录是 MySQL 用户的默认家目录，需要手动创建并设置正确权限：
	
	```
	sudo mkdir /nonexistent
	sudo chown mysql:mysql /nonexistent
	```
	
	- 重新启动 MySQL 服务
	在 WSL 中，使用 `service` 命令而不是 `systemctl`：
	
	```
	sudo service mysql stop
	sudo service mysql start
	```
	
	- 检查服务状态
	
	```
	sudo service mysql status
	```
	
	如果显示 `mysql start/running, process XXXX`，说明服务已成功启动。
	
	- 尝试登录
	
	服务启动后，使用无密码方式登录：
	
	```
	sudo mysql -u root
	```
	
**确认 MySQL 服务的监听地址**

```shell
# 查看 MySQL 监听的 IP 和端口 
sudo netstat -tulpn | grep mysqld
```
- 如果输出只有 `127.0.0.1:3306` 或 `localhost:3306`：说明 MySQL 只监听本地，拒绝所有远程连接；
- 如果输出有 `0.0.0.0:3306`：说明监听所有 IP，问题出在防火墙 / 权限。


# 安装redis-server

### 第一步：先安装 Redis（如果未安装）

在 WSL 终端执行以下命令：

```Bash
# 更新软件源
sudo apt update
# 安装 Redis 服务器
sudo apt install redis-server -y
```

### 第二步：启动 Redis 服务

WSL 中不支持 `systemctl`，用 `service` 命令启动：

```Bash
# 启动 Redis
sudo service redis-server start

# 可选：停止 Redis（需要时用）
# sudo service redis-server stop

# 可选：重启 Redis（配置修改后用）
# sudo service redis-server restart
```

### 第三步：验证 Redis 是否启动成功

```Bash
# 1. 查看服务状态（显示 running 则成功）
sudo service redis-server status

# 2. 连接 Redis 客户端测试
redis-cli

# 进入客户端后执行 ping，返回 PONG 则正常
127.0.0.1:6379> ping
PONG

# 退出客户端
127.0.0.1:6379> exit
```

### 第四步：（可选）设置 Redis 开机自启

WSL 重启后 Redis 会停止，可设置自启（两种方式选其一）：

#### 方式 1：通过 rc.local 配置（推荐）

```Bash
# 创建/编辑 rc.local 文件
sudo nano /etc/rc.local

# 在文件中添加以下内容（保存退出）
#!/bin/bash
sudo service redis-server start
exit 0

# 赋予执行权限
sudo chmod +x /etc/rc.local
```

#### 方式 2：通过 .bashrc 配置（每次终端启动都触发）

```Bash
# 编辑 bashrc 文件
nano ~/.bashrc

# 在文件末尾添加
sudo service redis-server start > /dev/null 2>&1

# 生效配置
source ~/.bashrc
```

### 第五步：（可选）允许远程连接 Redis（和 MySQL 同理）

默认 Redis 只监听 [127.0.0.1](127.0.0.1)，如需远程连接：

```Bash
# 编辑 Redis 配置文件
sudo nano /etc/redis/redis.conf

# 1. 注释 bind 127.0.0.1 ::1（加 #）
# bind 127.0.0.1 ::1

# 2. 关闭保护模式（把 protected-mode yes 改成 no）（仅测试环境） 
protected-mode no

# 3. 以守护进程运行
daemonize yes 

# 保存退出后重启 Redis
sudo service redis-server restart
```

### 总结

1. WSL 中启动 Redis 核心命令是 `sudo service redis-server start`；
    
2. 验证启动用 `redis-cli ping`，返回 PONG 即成功；
    
3. 如需远程连接，需修改 Redis 配置文件（注释 bind + 关闭保护模式）；
    
4. 可通过 rc.local 或 .bashrc 设置开机自启，避免每次重启 WSL 都手动启动。
